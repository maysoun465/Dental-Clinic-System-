Project Overview

The Dental Clinic System is a web-based application designed to manage basic clinic operations. The system supports different user roles—patient, doctor, and receptionist—each with specific access and functionality. The system simplifies appointment booking, patient management, and financial tracking. The project is partially implemented.

Technology Stack

Backend: .NET

Frontend: React

Architecture: Web-based client–server system

System Users and Roles

Patient – Access to general information, services, contact, dashboard, and appointment booking.

Doctor – Access to doctor dashboard, patient information, appointments, and finance section.

Receptionist – Manage appointments and access finance page.

Each user accesses the system after authentication and is redirected based on their role.

Functional Requirements
1. User Registration and Authentication

Sign-up page for new users.

Personal information entry during registration.

Login page for registered users.

Role selection (patient, doctor, receptionist) during login.

2. Patient Functionalities

About Us page.

Services page.

Contact page.

Patient Dashboard.

Book appointments.

Store patient appointment requests.

3. Doctor Functionalities

Doctor Dashboard.

View patient information.

View appointments.

Finance section.

4. Receptionist Functionalities

View and manage appointments.

Finance page.

Non-Functional Requirements
Usability

Simple and clear user interface.

Easy navigation for all roles.

Performance

Quick response to user actions.

Support multiple concurrent users.

Security

Access restriction based on user roles.

Secure user data during authentication.

Maintainability

Modular system for future improvements.

Easy addition of new features.

System Constraints

Assumptions

Limitations
